## [0:00–0:30] เปิดตัว — ปัญหา (Hook)

**สวัสดีคณะกรรมการทุกท่านครับ/คะ**

ผม/ดิฉันขอเริ่มด้วยคำถามง่ายๆ — ในองค์กรของเราเวลานี้ **"งาน" กับ "คน"** มันเชื่อมถึงกันจริงหรือ?

เรามีระบบงาน (Task/Project Management) อันหนึ่ง มีระบบเรียนรู้ (LMS) อีกอันหนึ่ง แต่ทั้งคู่ไม่เคยคุยกัน ผลคือ — คนทำงานที่ไม่มีทักษะพอ ได้งานไปทำแล้วพลาดเป้า หรือคนที่เรียนจบหลักสูตรเก่งๆ กลับไม่ได้รับงานที่ใช่

**เราสร้าง "สะพาน" เชื่อมสองโลกนี้ด้วย AI**

## [0:30–1:30] โซลูชัน — ภาพรวมระบบ

นี่คือ **AI-Driven Task & Learning Management Ecosystem** — ระบบนิเวศบริหารงานและพัฒนาบุคลากรที่เอาข้อมูลดิบ JSON มาตรฐานขององค์กร มาเปลี่ยนเป็นแดชบอร์ดอัจฉริยะในเบราว์เซอร์เดียว

ไฮไลต์ 3 ฟีเจอร์ที่จะโชว์วันนี้:

1. **Skill Gap Analyzer**
2. **AI Agent เสนอโครงการ**
3. **AI Chatbot + Failover Router**

ก่อนเข้าแต่ละฟีเจอร์ ขออธิบายโครงสร้างระบบคร่าวๆ:

- เป็น **Client-side PWA** 100% — ไม่ต้องมี Server ก็ทำงานได้
- ใช้ **Vanilla JS + Chart.js + WebGL**
- ออกแบบภายใต้คอนเซปต์ **Liquid Glass** — กระจกฝ้า โปร่งแสง 3D Tilt ให้ความรู้สึกพรีเมียม ทันสมัย

## [1:30–3:00] Feature 1: Dashboard & Skill Gap Analyzer (Live Demo)

**ขออนุญาตเปิดหน้าจอระบบจริงครับ**

**(เปิดแอป — ชี้ให้เห็น Dashboard)**

เรามีข้อมูลพนักงานจาก `users.json`, งานจาก `tasks.json`, คอร์สจาก `courses.json` และประวัติลงทะเบียนเรียนจาก `enrollments.json` — รวมกันในหน้าเดียว

**(ชี้ KPI Metrics)**

บนนี้คือ KPI หลัก: จำนวนงานสำเร็จ, อัตราความเสี่ยง, ทักษะที่พัฒนาแล้ว — Animation วิ่งนับให้เห็นภาพ

**(คลิกเข้าไปที่งานหรือคอร์ส)**

เมื่อคลิก — Modal กระจกฝ้าจะแสดงรายละเอียดเชิงลึก พร้อมแนะนำพนักงานที่มีทักษะใกล้เคียงกัน โดยระบบจะเชื่อมโยงว่า — คนนี้เรียนคอร์สอะไรไปแล้ว? งานที่ได้รับตรงกับ skillset หรือเปล่า? ถ้าไม่ตรง ระบบจะ **Flag เตือน** ทันที

นี่คือ **Skill Gap Analyzer** — หัวใจสำคัญที่ทำให้ "งาน" และ "คน" เชื่อมถึงกัน

## [3:00–4:30] Feature 2: AI Agent & Sub-Project Generator (Live Demo)

**(เปิด AI Agent Panel)**

ต่อไปเป็นฟีเจอร์ที่ผม/ดิฉันชอบที่สุด — **AI Agent**

ระบบจะสแกนข้อมูลทั้งหมดที่มี ทั้งพนักงาน งานค้าง ทักษะที่มีและขาด จากนั้นส่ง prompt ไปยัง AI เพื่อให้ **เสนอ 3 โครงการย่อย (Sub-projects)** ที่ช่วยสร้างสมดุลให้องค์กร เช่น:

- โครงการที่ใช้คนที่มีภาระงานน้อย
- โครงการที่ตรงกับทักษะที่เพิ่งเรียนจบมา
- โครงการที่ช่วยลดความเสี่ยงของโครงการหลักที่ใกล้เดดไลน์

**(คลิก Generate — รอผล)**

AI จะสร้างการ์ดขึ้นมา 3 ใบ แต่ละใบมี: ชื่อโครงการ, พี่เลี้ยงที่แนะนำ, และทักษะที่เกี่ยวข้อง

**(คลิก Approve บนการ์ดใบหนึ่ง)**

เมื่อกด **Approve** — ระบบจะบันทึกงานใหม่เข้า LocalStorage และสะท้อนไปยัง Dashboard, KPI, Charts ทันทีแบบ Real-time โดยไม่ต้องรีเฟรชหน้า

## [4:30–6:00] Feature 3: AI Chatbot + Failover Router (Live Demo)

**(เปิด Chatbot Drawer ด้านขวา)**

เรามี **AI Chatbot** ที่สามารถถาม-ตอบเกี่ยวกับภาพรวมโครงการ หรือขอคำแนะนำทักษะได้

ลองถาม: *"โปรเจกต์ไหนมีความเสี่ยงมากที่สุด?"*

**(พิมพ์ถาม — รอคำตอบ)**

ระบบตอบกลับภายใน 0.5 วินาที ผ่าน **Groq/Llama 3** — เร็วพอที่จะสนทนาต่อเนื่องได้เป็นธรรมชาติ

แต่ที่เจ๋งกว่าคือเบื้องหลัง — **Multi-API Failover Router**

**(อธิบาย Architecture)**

ระบบมีการจัดเส้นทาง 4 ชั้น:
1. **Gemini 2.5 Flash** — ตัวหลักสำหรับงานวิเคราะห์เชิงลึก
2. **OpenRouter (Free)** — สำรองชั้น 1
3. **Groq Cloud (Llama 3)** — หลักสำหรับ Chatbot / สำรอง Analysis
4. **Local Rule-based Simulator** — สำรองสุดท้าย ทำงานในเครื่อง 100% ไม่ต้องพึ่ง API ภายนอก

ข้อดี: แอป **ล่มไม่ได้** — ต่อให้ทุก API ล้ม ระบบก็ยังตอบได้ด้วย Simulator ในเครื่อง เพราะเราให้ความสำคัญกับ **Reliability** ที่องค์กรต้องการจริงๆ

## [6:00–7:30] โบนัส: PWA, Real-time Simulator, Premium UI

นอกจากฟีเจอร์หลัก เราเพิ่มเติมประสบการณ์ให้เหนือระดับ:

**PWA + Offline-First:**
- ติดตั้งแอปบนมือถือได้เหมือน Native App ผ่าน manifest.json
- Service Worker แคชไฟล์ทั้งหมด — เปิดใช้งานได้แม้ไม่มีอินเทอร์เน็ต

**Live Simulator:**
- ด้านล่าง Sidebar มี Toggle — เมื่อเปิด ระบบจะจำลองข้อมูลใหม่ (เดดไลน์กระชั้น, งานเพิ่ม, ความก้าวหน้า) ไหลเข้าสู่ Dashboard แบบ Real-time
- มี **Activity Ticker** ที่บอกสถานะล่าสุด ค่อยๆ เลื่อนด้านล่าง

**Premium UI:**
- Liquid Glass — หน้าต่างโปร่งแสงซ้อนทับ สวยงาม ทันสมัย
- WebGL Gold Wave — พื้นหลังคลื่นทองเคลื่อนไหว
- Sparkle Trail — ละอองดาวตามเมาส์
- 3D Tilt Card — การ์ดเอียงตามตำแหน่งเมาส์ ให้มิติ 3D

## [7:30–8:30] Tech Stack & สถาปัตยกรรมสรุป

**(สรุปแบบรวดเร็ว)**

```text
Client: HTML5 + CSS3 (Liquid Glass) + Vanilla JS (ES6+)
Charts: Chart.js (Doughnut, Bar)
Background: WebGL Shader (Custom GLSL)
Storage: Browser LocalStorage + JSON seed data
AI Router: Gemini → OpenRouter → Groq → Local Simulator
PWA: manifest.json + sw.js (Offline Cache)
```

สถาปัตยกรรมแบบ **Client-side เท่านั้น** — ไม่ต้องพึ่ง Backend Server ทำให้ deploy ง่าย, ถูก, และเร็ว

## Bonus: คำถามที่คาด (Q&A Prep)

**Q: ทำไมต้องเป็น Client-side ทั้งหมด?**
A: เพื่อให้ลงสนามได้เร็วที่สุด ไม่ต้องพึ่ง IT ตั้ง Server เปิดใน Browser ก็ใช้ได้ทันที

**Q: ข้อมูลพนักงานเยอะมากจะช้าหรือเปล่า?**
A: เราออกแบบให้ใช้ JSON + LocalStorage ที่โหลดครั้งเดียว และใช้ In-memory Processing

**Q: AI Failover ทำงานยังไง?**
A: ใช้ Try-Catch chain — ถ้าตัวแรก Error (เช่น Rate limit / Invalid key) จะข้ามไปตัวถัดไปทันที โดยไม่สะดุด

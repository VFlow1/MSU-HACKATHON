### ช่วงที่ 2: แนะนำโปรเจกต์ & แนวคิดหลัก (0:25–1:20)

**🎥 หน้าจอ:** ค่อยๆ Pan/Scroll ให้เห็นภาพรวมหน้า Dashboard ทั้งหมด — Sidebar ด้านซ้าย (โลโก้ AI-TALENT รูปสมอง), Topbar, และ KPI Cards แถวบน

**🗣️ บทพูด:**
> "AI-TALENT (เอ-ไอ-ทาเลนต์) ย่อมาจาก *AI-Driven Task & Learning Management Ecosystem (เอ-ไอ-ไดรฟ์เวิน ทาสก์ แอนด์ เลิร์นนิ่ง แมเนจเมนต์ อีโคซิสเต็ม)* เป็น Client-side Web Application (ไคลเอนต์-ไซด์ เว็บ แอปพลิเคชัน) ที่ชุบชีวิตข้อมูล JSON (เจสัน) ดิบๆ ของบุคลากร โปรเจกต์ งาน คอร์สเรียน และการลงทะเบียน ให้กลายเป็นระบบนิเวศเดียวที่บริหารทั้ง **'การส่งงานให้เสร็จ' (Task Delivery - ทาสก์ ดีลิเวอรี่)** และ **'การพัฒนาทักษะคน' (Upskilling - อัปสกิลลิ่ง)** ไปพร้อมกัน โดยมี AI (เอ-ไอ) เป็นตัวเชื่อมสามจุดหลักครับ"

> "หนึ่ง — **3-Month Skill Gap Analyzer (ทรี-มันธ์ สกิล แกป อะนาไลเซอร์)** คาดการณ์ล่วงหน้า 3 เดือนจาก Roadmap (โรดแมป) งานที่กำลังเข้ามา เปรียบเทียบกับทักษะปัจจุบันของพนักงานในทีม และแนะนำคอร์สเรียน LMS (แอล-เอ็ม-เอส) ที่สอดคล้องเพื่ออุดรอยรั่วทักษะทันที"

> "สอง — **AI Agent (เอ-ไอ เอเจนต์)** ที่ระดมไอเดียโปรเจกต์ย่อย พร้อมจับคู่คนที่มีทักษะตรงที่สุดให้อัตโนมัติ"

> "และสาม — **AI Failover Router (เอ-ไอ เฟลโอเวอร์ เอาต์เตอร์)** ระบบจัดเส้นทาง AI (เอ-ไอ) ถึง 4 ชั้น สลับจาก Gemini (เจมินาย) ไป OpenRouter (โอเพน รูทเตอร์) ไป Groq (กรอค) จนถึง Local Simulator (โลคอล ซิมูเลเตอร์) เพื่อการันตีว่าแอปนี้ใช้งานได้ 100% ตลอดเวลา ไม่ว่าคีย์ AI (เอ-ไอ) จะมีหรือไม่มีก็ตาม"

> "เดี๋ยวเราไปดูของจริงกันเลยครับ"

### ช่วงที่ 3: Dashboard Overview Demo (1:20–2:50)

**🎥 หน้าจอ — ลำดับการกระทำ:**
1. อยู่ที่หน้า **Dashboard** (หน้าแรก) — ขยับเมาส์ผ่าน KPI Cards ช้าๆ ให้เห็นเอฟเฟกต์ **3D Tilt + แสง Glare** สะท้อนตามเมาส์
2. ปล่อยให้ตัวเลข KPI **Count-Up Animation** วิ่งขึ้น (ถ้ายังไม่ขึ้น ให้ Refresh หน้าก่อนอัด)
3. ขยับเมาส์ผ่านพื้นที่ Dashboard ให้เห็น **Sparkle Trail** สีทองปลิวตาม
4. เลื่อนลงไปที่ **Live Activity Ticker** ให้เห็นข้อความเลื่อนวิ่ง
5. ชี้ที่ **AI Banner** "AI ตรวจพบความเสี่ยง 3 รายการ"
6. เลื่อนลงไปที่กราฟ Doughnut + Bar Chart (แนวโน้มประสิทธิภาพ / สถานะงาน)
7. เลื่อนลงสุดไปที่ตาราง "งานที่ต้องติดตามด่วน" พร้อม Pagination

**🗣️ บทพูด:**
> "นี่คือหน้า Dashboard (แดชบอร์ด) ภาพรวมครับ — สิ่งแรกที่จะสังเกตได้คือธีมที่เราเรียกว่า **'Liquid Glass' (ลิควิด กลาส)** พื้นหลังเป็น WebGL Shader (เว็บ-จี-แอล เชดเดอร์) คลื่นแสงสีทองที่เคลื่อนไหวตลอดเวลา ส่วน Panel (พาเนล) ทุกชิ้นเป็นกระจกฝ้าโปร่งแสงแบบ Glassmorphism (กลาสมอร์ฟิซึม) ซ้อนทับกันอย่างมีมิติ"

> "การ์ด KPI (เค-พี-ไอ) ด้านบนนี้ไม่ใช่แค่ตัวเลขนิ่งๆ นะครับ — ลองดูตอนผมขยับเมาส์ การ์ดจะ **เอียงตามมุมมอง 3 มิติ** พร้อมแสงสะท้อน Glare (แกลร์) วิบวับ และตัวเลขจะ **วิ่งนับขึ้นจาก 0** แบบ Ease-out (อีส-เอาต์) ทุกครั้งที่โหลดหน้า — เราติดตามงานที่เสร็จแล้ว 139 งาน งานที่ติดขัด 15 รายการ อัตราความก้าวหน้า LMS (แอล-เอ็ม-เอส) 59.3% และคะแนนประสิทธิภาพทีมโดยรวม 42.5"

> "และนี่ครับ — ละอองดาวสีทองที่ปลิวตามเมาส์ของผม คือ **Sparkle Trail Effect (สปาร์กเกิล เทรล เอฟเฟกต์)** เป็นหนึ่งใน Premium Interaction (พรีเมียม อินเตอร์แอคชัน) ที่เราใส่ใจในรายละเอียด UI/UX (ยู-ไอ / ยู-เอ็กซ์)"

> "เลื่อนลงมา จะเห็น **Live Activity Ticker (ไลฟ์ แอคทิวิตี ทิกเกอร์)** — แถบนี้จะอัปเดตกิจกรรมล่าสุดของระบบแบบ Real-time (เรียลไทม์) จากตัวจำลองข้อมูลที่ทำงานอยู่เบื้องหลัง ไม่ต้องกด Refresh (รีเฟรช) เลย"

> "ส่วนแบนเนอร์สีทองนี้คือ **AI Risk Banner (เอ-ไอ ริสก์ แบนเนอร์)** — ระบบตรวจพบว่างาน 'Setup CI/CD Pipeline' (เซ็ตอัป ซี-ไอ/ซี-ดี ไปป์ไลน์) ใกล้ถึง Deadline (เดดไลน์) แต่คนที่รับผิดชอบยังเรียนคอร์ส Cloud (คลาวด์) ไม่จบ ระบบจึงแจ้งเตือนทันทีพร้อมปุ่มดูรายละเอียดและอนุมัติ"

> "ด้านล่างคือกราฟแสดงแนวโน้มประสิทธิภาพงานเทียบกับงานที่ติดขัด และสัดส่วนสถานะงานทั้งหมด — เชื่อมข้อมูลจากฐานข้อมูลจริงแบบ Real-time (เรียลไทม์) ทุกจุดครับ"

### ช่วงที่ 4: Tasks / LMS / Employees — List, Search, Filter, Detail (2:50–4:10)

**🎥 หน้าจอ — ลำดับการกระทำ:**
1. คลิกเมนู **"จัดการงาน (Tasks)"** ที่ Sidebar
2. พิมพ์คำค้นหาในช่อง Search เช่น "API" เพื่อกรองตาราง
3. เปลี่ยน Filter Dropdown เป็น "ติดขัด" (blocked) ให้เห็นผลกรองทันที
4. **คลิกที่แถวงาน 1 แถว** → ระบบเปิด **Detail Page Modal** กระจกฝ้า แสดงรายละเอียดงาน + AI Risk + คำแนะนำพนักงานทดแทน
5. คลิก "กลับไปหน้าก่อนหน้า" แล้วไปที่เมนู **"ระบบเรียนรู้ (LMS)"**
6. ชี้คอลัมน์ "AI แนะนำ" ในตารางคอร์ส
7. ไปที่เมนู **"บุคลากร"** คลิกพนักงาน 1 คน → ดู Detail (ทักษะ, งานที่รับผิดชอบ, คอร์สที่เรียน, Performance Score)
8. กดปุ่ม **"Export CSV"** ให้เห็นไฟล์ถูกดาวน์โหลด

**🗣️ บทพูด:**
> "มาที่หน้า **จัดการงาน** ครับ — นี่คือตารางงานทั้งหมดในระบบ แสดงชื่องาน ผู้รับผิดชอบ ฝ่าย ระดับความสำคัญ สถานะ และ Progress Bar (โปรเกรส บาร์) ผมสามารถพิมพ์ค้นหาชื่องานหรือชื่อพนักงานได้ทันที และกรองตามสถานะ เช่น เลือกดูเฉพาะงานที่ 'ติดขัด' — เห็นไหมครับ ตารางกรองให้ทันที"

> "ทีนี้ลองคลิกที่งานหนึ่งรายการ... ระบบจะเปิด **หน้ารายละเอียดแบบ Modal (โมดอล) กระจกฝ้า** ขึ้นมา แสดงข้อมูลเชิงลึกทั้งหมดของงานนั้น พร้อมระดับความเสี่ยงที่ AI (เอ-ไอ) ประเมินไว้ และถ้าทักษะของคนที่รับงานไม่ตรง ระบบจะแนะนำพนักงานที่มีทักษะใกล้เคียงให้ทันทีเช่นกัน"

> "ย้ายไปที่ **ระบบเรียนรู้ (LMS - แอล-เอ็ม-เอส)** — ตรงนี้คือคอร์สเรียนทั้งหมดที่พนักงานลงทะเบียนไว้ มี Progress (โปรเกรส) การเรียน จำนวนบทเรียนที่เหลือ และคอลัมน์สุดท้ายคือ **'AI (เอ-ไอ) แนะนำ'** ซึ่งจะปรับคำแนะนำตามสถานะ — ถ้าเรียนค้างนานก็แนะนำให้แบ่งเป็นชิ้นเล็กๆ วันละ 15 นาที ถ้าเรียนจบแล้วก็แนะนำคอร์สขั้นสูงถัดไป"

> "และในหน้า **บุคลากร** ผมสามารถคลิกดูโปรไฟล์รายบุคคล เห็นทักษะหลัก งานที่รับผิดชอบ คอร์สที่เรียน และคะแนนประสิทธิภาพที่คำนวณจากอัตราการทำงานเสร็จรวมกับความก้าวหน้าการเรียนแบบถ่วงน้ำหนัก — ทุกตารางในระบบยังกด **Export CSV (เอ็กซ์พอร์ต ซี-เอส-วี)** ออกไปใช้ต่อได้เลยครับ"

### ช่วงที่ 8: Liquid Glass UI Recap + Tech Stack/Architecture (7:50–8:35)

**🎥 หน้าจอ — ลำดับการกระทำ:**

3. ไปที่เมนู **"ตั้งค่า"** ชี้สถานะการเชื่อมต่อ AI 3 ค่าย (Gemini / Groq / OpenRouter)
4. ชี้ Toggle "เปิดใช้งานการจำลองข้อมูลไหลเข้า" และตัวเลือกความถี่

**🗣️ บทพูด:**
> "สรุปภาพรวมด้าน UI/UX (ยู-ไอ / ยู-เอ็กซ์) อีกครั้ง — ทั้งระบบใช้ธีม **Liquid Glass (ลิควิด กลาส)** พื้นหลัง WebGL Shader (เว็บ-จี-แอล เชดเดอร์) เคลื่อนไหวตลอดเวลา ทุก Panel (พาเนล) เป็นกระจกฝ้าโปร่งแสงแบบ `backdrop-filter: blur()` ซ้อนกันหลายชั้น เรายังล็อกธีมเป็น Dark Mode (ดาร์ก โหมด) ถาวรเพื่อความหรูหราและลื่นไหล ไม่มีปัญหาแสงกะพริบตอนสลับธีม"

> "ด้านเทคนิค ฝั่งหน้าบ้านเราใช้ HTML5 (เอช-ที-เอ็ม-แอล ไฟฟ์), Vanilla CSS3 (วานิลลา ซี-เอส-เอส ทรี) และ JavaScript ES6+ (จาวาสคริปต์ อี-เอส ซิกส์ พลัส) ล้วนๆ — ไม่มี Framework (เฟรมเวิร์ก) หนักๆ ใช้ Chart.js (ชาร์ต-เจ-เอส) สำหรับกราฟ และ FontAwesome (ฟอนต์-ออซัม) สำหรับไอคอน ส่วนพื้นหลัง WebGL เขียนด้วย Shader ของเราเอง"

> "ในหน้าตั้งค่า เราเปิดให้เห็นสถานะการเชื่อมต่อ AI (เอ-ไอ) ทั้ง 3 ค่ายแบบ Real-time (เรียลไทม์) — Gemini (เจมินาย), Groq (กรอค), และ OpenRouter (โอเพน รูทเตอร์) — และยังมีระบบจำลองข้อมูล Real-time (เรียลไทม์) ที่ปรับความถี่ได้ ตั้งแต่ 5 วินาทีไปจนถึง 60 วินาที เพื่อทดสอบว่าระบบ Live Ticker (ไลฟ์ ทิกเกอร์) และ KPI (เค-พี-ไอ) อัปเดตสดได้จริง"

> "และที่สำคัญ เราออกแบบให้ผ่านเกณฑ์ **Lighthouse Accessibility (ไลท์เฮาส์ แอคเซสซิบิลิตี) และ Performance (เพอร์ฟอร์แมนซ์) ≥ 90** ด้วยการเพิ่ม aria-label ให้ทุกปุ่มควบคุม รองรับ Responsive (เรสปอนซิฟ) ทั้ง Mobile (โมบาย) และ Desktop (เดสก์ท็อป) ครับ"

## 🎤 เทคนิคการพรีเซนต์ (Delivery Tips)

- **น้ำเสียง:** ช่วงเปิด (Hook) ให้พูดช้าและหนักแน่น สร้างความรู้สึก "ปัญหานี้จริง" ส่วนช่วง Demo ให้พูดกระชับ มั่นใจ เหมือนกำลังโชว์ของที่ภูมิใจ
- **จังหวะคลิก:** อย่าคลิกเร็วเกินคำพูด — เผื่อเวลาให้ Animation (Count-up, Loading 4-step, AI พิมพ์ตอบ) เล่นจบก่อนพูดประโยคต่อไป เพราะ Animation เหล่านี้คือจุดขาย UI/UX
- **เสียงเอฟเฟกต์:** ถ้าจะโชว์เสียง Chime ให้เปิด Sound Toggle ไว้ตั้งแต่ต้นวิดีโอ และตรวจสอบว่าไมค์ไม่หยิบเสียงรบกวนจากลำโพงคอม
- **ลำดับ Cursor:** ฝึกเส้นทางการคลิกล่วงหน้าให้ลื่น (Dashboard → Tasks → LMS → Employees → AI Insights → AI Agent → Chatbot → Offline → Reports → Settings) จะช่วยให้ไม่หลงทางตอนอัดจริง
- **คำเชื่อม:** ใช้คำเชื่อมสั้นๆ เช่น "มาดูกันต่อ" / "อีกฟีเจอร์หนึ่งคือ" / "และที่เจ๋งกว่านั้นคือ" เพื่อให้การเปลี่ยนหน้าไม่สะดุด

- [ ] ตรวจว่ามีคีย์ AI อย่างน้อย 1 ค่าย (Gemini/Groq/OpenRouter) ใน `.env` เพื่อให้ช่วงที่ 5 และ 6 แสดงผลวิเคราะห์ "สด" จาก AI จริง — ถ้าไม่มีคีย์เลย ระบบจะ Fallback ไป Local Simulator ซึ่งยังใช้งานได้ แต่คำตอบจะไม่ Dynamic เท่า
- [ ] เคลียร์ `localStorage` (กดปุ่ม "รีเซ็ตข้อมูลทั้งหมด" ในหน้าตั้งค่า) ก่อนอัด เพื่อให้ KPI/ตัวเลขเริ่มต้นตรงกับที่อยู่ในสคริปต์
- [ ] ทดสอบ DevTools → Network → Offline ล่วงหน้าหนึ่งรอบ ให้แน่ใจว่า Service Worker แคชสำเร็จก่อนปิดเน็ตจริงตอนอัด

## ⏱️ การปรับความยาว (Flexible Timing Guide)

### ถ้าต้องตัดให้เหลือ ~5 นาที (ตามเกณฑ์โจทย์)
ตัด/รวบช่วงเหล่านี้:
- รวมช่วงที่ 1+2 ให้เหลือ ~40 วินาที (พูดปัญหา + แนวคิดในประโยคเดียวกัน)
- ช่วงที่ 3 (Dashboard) ตัดเหลือ ~50 วินาที — โฟกัสที่ AI Banner + กราฟ ข้ามการอธิบาย Sparkle Trail แบบละเอียด (แค่ปล่อยให้เห็นในภาพพอ)
- ช่วงที่ 4 (Tasks/LMS/Employees) ตัดเหลือ ~40 วินาที — โชว์ Search/Filter + Detail Modal ของ Tasks อย่างเดียว ข้าม LMS/Employees ไปพูดถึงสั้นๆ ด้วยคำพูด
- ช่วงที่ 5 (AI Insights + Agent) **คงไว้เต็ม ~90 วินาที** เพราะเป็นจุดขายหลัก — ตัดได้แค่ Timeline ประวัติ AI ออก
- ช่วงที่ 6 (Chatbot) ตัดเหลือ ~25 วินาที — โชว์แค่ 1 คำถามพอ
- ช่วงที่ 7 (PWA/Reports) เลือกโชว์แค่ 1 อย่าง (แนะนำ Offline Demo เพราะ "เห็นภาพ" ชัดกว่า) ตัด Reports ออกพูดด้วยคำพูดแทน
- ช่วงที่ 8 ตัดเหลือ 1 ประโยคสรุปสั้นๆ ผนวกกับช่วงปิดท้าย

### ถ้าจะยืดเป็น ~10 นาที (มีเวลาเผื่อ Q&A)
- เพิ่ม Use Case สมมติในช่วงที่ 5: ลองพิมพ์โจทย์ AI Agent อีกแบบ เช่น "วางแผนยกระดับเรียนรู้ LMS แก้ปัญหางานสะดุด (Remedial Plan)" เพื่อโชว์ว่า Agent ปรับผลตาม Input ได้จริง ไม่ได้ Hardcode
- เพิ่มการอธิบาย `userMetaMap` และโครงสร้างไฟล์ `data/*.json` สั้นๆ ในช่วงที่ 4 เพื่อโชว์ Data Handling
- เพิ่ม Mermaid Diagram สถาปัตยกรรม AI Router (จาก `ARCHITECTURE.md`) เป็นภาพ Overlay ในช่วงที่ 8 พร้อมอธิบายเส้นทาง Failover ทีละขั้น
- เปิดท้ายวิดีโอด้วยช่วง "คำถามที่กรรมการอาจถาม" 1-2 ข้อ พร้อมคำตอบสั้นๆ (เช่น "ถ้า API ทุกตัวล่มพร้อมกันจะเกิดอะไรขึ้น" → อธิบาย Local Rule-based Simulator)
